# Emotion-LLaMA 特征提取 · AutoDL 完整操作步骤

> 本文档仅包含操作步骤，脚本原理与架构说明见 [`特征提取脚本说明.md`](./特征提取脚本说明.md)。

---

## 第一步：创建 AutoDL 实例

1. 登录 [AutoDL 控制台](https://www.autodl.com/console/instance/list)
2. 点击「租用新实例」，选择配置：
   - **GPU**：RTX 4090（24GB）× 1，或 RTX 3090（24GB）× 1
   - **镜像**：`PyTorch 2.0.0 / Python 3.10 / CUDA 11.8`
3. **存储**说明：
   - `/root/autodl-tmp`（系统盘，500GB，读写快，**关机不保留**，放数据集和权重）
   - `/root/autodl-fs`（文件存储，按需挂载，关机保留，放输出特征）
   - 建议：在「存储 → 文件存储」处新建一个存储并挂载到实例
4. 点击「立即租用」，等待实例启动后点击「JupyterLab」或使用 SSH 连接

## 第二步：上传数据集

### 方式一：AutoDL 文件上传（推荐，适合 <20GB）

```bash
# 本地打包后上传（在本地终端执行）
tar -czf CA-MER.tar.gz CA-MER/
# 在 JupyterLab 左侧文件树中上传 CA-MER.tar.gz 到 /root/autodl-tmp/datasets/
# 然后在 AutoDL 终端中解压：
cd /root/autodl-tmp/datasets && tar -xzf CA-MER.tar.gz
```

### 方式二：AutoDL 数据集市场 / 对象存储（适合大数据集）

```bash
# 若使用 AutoDL 数据集中转，按控制台提示操作
# 最终确保目录结构如下：
/root/autodl-tmp/datasets/
├── CA-MER/
│   ├── video-aligned.json       ← 标注文件
│   └── video-aligned/           ← 视频文件目录
│       ├── sample_00000001.avi
│       └── ...
└── MER2025-ovmer/               ← EMER 数据集（如有）
```

## 第五步：下载模型权重

### 5.1 Emotion-LLaMA 微调权重（手动下载，922 MB）

浏览器打开：https://huggingface.co/spaces/ZebangCheng/Emotion-LLaMA/tree/main

找到 `Emoation_LLaMA.pth`（注意：原文件名含拼写错误），点击下载图标 ↓。

下载后上传至 AutoDL（同第二步上传方式），放到：
```
/root/autodl-tmp/checkpoints/Emoation_LLaMA.pth
```

也可在 AutoDL 终端直接用 `wget` 下载（需要网络能访问 HuggingFace）：
```bash
mkdir -p /root/autodl-tmp/checkpoints
# 方式一：直接 wget（需能访问 HuggingFace）
wget -O /root/autodl-tmp/checkpoints/Emoation_LLaMA.pth \
  "https://huggingface.co/spaces/ZebangCheng/Emotion-LLaMA/resolve/main/Emoation_LLaMA.pth"

# 方式二：使用学术加速（AutoDL 学术资源加速）
source /etc/network_turbo  # 开启加速
wget -O /root/autodl-tmp/checkpoints/Emoation_LLaMA.pth \
  "https://huggingface.co/spaces/ZebangCheng/Emotion-LLaMA/resolve/main/Emoation_LLaMA.pth"
```

### 5.2 EVA-CLIP-G 基础权重（首次运行自动下载，~3.6 GB）

**无需手动操作。** 首次运行特征提取脚本时，代码会自动从 Google Cloud Storage 下载并缓存：
```
下载地址：https://storage.googleapis.com/.../eva_vit_g.pth
本地缓存：~/.cache/torch/hub/checkpoints/eva_vit_g.pth
```

若网络受限，也可以提前手动下载：
```bash
mkdir -p ~/.cache/torch/hub/checkpoints
# 开启 AutoDL 学术加速后下载
source /etc/network_turbo
wget -O ~/.cache/torch/hub/checkpoints/eva_vit_g.pth \
  "https://storage.googleapis.com/sfr-vision-language-research/LAVIS/models/BLIP2/eva_vit_g.pth"
```

### 5.3 权重汇总

| 文件 | 大小 | 存放路径 | 是否需要手动下载 |
|------|------|---------|--------------|
| `Emoation_LLaMA.pth` | 922 MB | `/root/autodl-tmp/checkpoints/` | ✅ 需要 |
| `eva_vit_g.pth` | ~3.6 GB | `~/.cache/torch/hub/checkpoints/` | 自动（或手动预下载）|

## 第六步：修改配置文件

```bash
cd /root/autodl-tmp/Emotion-LLaMA

# 用 nano 或 vim 编辑配置文件
nano feature_extraction_config_llama.yaml
```

**必须修改的字段：**

```yaml

  ckpt: "/root/autodl-tmp/checkpoints/Emoation_LLaMA.pth"  # 确认路径正确
  device: "cuda:0"

data:
  dataset_path: "/root/autodl-tmp/datasets/CA-MER"  # 改为实际数据集路径
  annotation_file: "video-aligned.json"              # 改为实际标注文件名
  video_dir: "video-aligned"                         # 改为视频子目录名

feature_saving:
  output_dir: "/root/autodl-tmp/features/EmotionLlamaEncoder"  # 输出路径

resume:
  checkpoint_file: "/root/autodl-tmp/logs/llama_feature_checkpoint.json"

logging:
  output_dir: "/root/autodl-tmp/logs/llama_features"
```

## 常见问题速查

| 问题现象 | 原因 | 解决方法 |
|---------|------|---------|
| `ConnectionError` 下载 eva_vit_g.pth | 网络无法访问 Google | `source /etc/network_turbo` 开启学术加速 |
| `CUDA out of memory` | 显存不足 | 配置文件中将 `max_frames` 从 16 改为 8 |
| `ImportError: No module named 'timm'` | 缺少依赖 | `pip install timm omegaconf peft` |
| `IOError: 无法打开视频` | 视频路径错误 | 检查 `video_dir` 配置是否与实际目录名匹配 |
| 日志显示 `⚠️ 未找到 ln_vision 权重` | checkpoint 路径错误 | 确认 `Emoation_LLaMA.pth` 路径正确（注意拼写）|
| inode 告警 | AutoDL 系统盘 inode 限制 | 将输出目录改到 `/root/autodl-fs`（文件存储）|
