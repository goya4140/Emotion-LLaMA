# Emotion-LLaMA 特征提取 · AutoDL 完整操作步骤

> 本文档仅包含操作步骤，脚本原理与架构说明见 [`特征提取脚本说明.md`](./特征提取脚本说明.md)。

---

## 第一步：创建 AutoDL 实例

1. 登录 [AutoDL 控制台](https://www.autodl.com/console/instance/list)
2. 点击「租用新实例」，选择配置：
   - **GPU**：RTX 4090（24GB）× 1，或 RTX 3090（24GB）× 1
   - **镜像**：`PyTorch 2.0.0 / Python 3.10 / CUDA 11.8`
3. **存储**说明：
   - `/root/autodl-tmp`（系统盘，读写快，**关机不保留**，放数据集和模型缓存）
   - `/root/autodl-fs`（文件存储，关机保留，放输出特征）
   - 建议：在「存储 → 文件存储」处新建存储并挂载到实例
4. 点击「立即租用」，等待实例启动后点击「JupyterLab」或使用 SSH 连接

## 第五步：模型权重说明

| 编码器 | 权重文件 | 大小 | 获取方式 |
|--------|---------|------|---------|
| EVA-CLIP-G | `eva_vit_g.pth` | ~3.6 GB | **首次运行自动下载**（缓存到 `~/.cache/torch/hub/checkpoints/`） |
| MAE-Large | HuggingFace `facebook/vit-mae-large` | ~1.3 GB | 首次运行自动下载 |
| VideoMAE-Large | HuggingFace `MCG-NJU/videomae-large` | ~1.3 GB | 首次运行自动下载 |

**网络受限时（开启 AutoDL 学术加速）：**

```bash
source /etc/network_turbo

# 手动预下载 EVA-CLIP 权重
mkdir -p ~/.cache/torch/hub/checkpoints
wget -O ~/.cache/torch/hub/checkpoints/eva_vit_g.pth \
  "https://storage.googleapis.com/sfr-vision-language-research/LAVIS/models/BLIP2/eva_vit_g.pth"

# HuggingFace 模型会在第一次 from_pretrained 时自动缓存到 ~/.cache/huggingface/
```

> 旧版操作指导中提到的 `Emoation_LLaMA.pth`（922MB）**不再需要**。
> 新版三路提取器均不加载 Emotion-LLaMA 投影层权重。

## 第六步：检查并修改配置文件

每个提取器有独立的配置文件，路径在 `feature_extractors/{encoder}/config_{encoder}.yaml`。

**必须确认的字段（以 EVA-CLIP 为例）：**

```yaml
# feature_extractors/eva_clip/config_eva_clip.yaml
datasets:
  - name: "CA-MER"
    dataset_path: "/root/autodl-tmp/datasets/CA-MER"   # 确认实际路径
    annotation_file: "video-aligned.json"
    video_dir: "video-aligned"

  - name: "EMER"
    dataset_path: "/home/user/Emotion-LLaMA"           # emotion_analysis_data.json 所在目录
    annotation_file: "emotion_analysis_data.json"
    video_dir: "video"                                  # 绝对路径时此字段作为回退

feature_saving:
  output_root: "/root/autodl-fs/features"              # 确认文件存储已挂载

resume:
  checkpoint_file: "/root/autodl-tmp/logs/eva_clip_{dataset}_checkpoint.json"

logging:
  output_dir: "/root/autodl-tmp/logs"
```

MAE 和 VideoMAE 配置结构相同，仅 `model.model_name` 和 `vision_preprocess` 不同。

## 第七步：运行特征提取

## 第八步：验证输出

```bash
python - <<'EOF'
import os, torch

output_root = "/root/autodl-fs/features"
for encoder in ["EVA-CLIP", "MAE", "VideoMAE"]:
    enc_dir = os.path.join(output_root, encoder)
    if not os.path.isdir(enc_dir):
        print(f"{encoder}: 目录不存在，跳过")
        continue
    for ds in sorted(os.listdir(enc_dir)):
        ds_dir = os.path.join(enc_dir, ds)
        files  = sorted(f for f in os.listdir(ds_dir) if f.endswith(".pt"))
        print(f"\n{encoder}/{ds}: 共 {len(files)} 个特征文件")
        for fname in files[:2]:
            d = torch.load(os.path.join(ds_dir, fname), map_location="cpu", weights_only=False)
            print(f"  {fname}: shape={list(d['features'].shape)}, dtype={d['features'].dtype}")

print("\n验证完成")
EOF
```

## 常见问题速查

| 问题现象 | 原因 | 解决方法 |
|---------|------|---------|
| `ConnectionError` 下载 eva_vit_g.pth | 网络无法访问 Google | `source /etc/network_turbo` 后手动下载 |
| `ConnectionError` 下载 HuggingFace 模型 | 网络受限 | `source /etc/network_turbo` 后再运行 |
| `CUDA out of memory`（EVA-CLIP） | 显存不足 | 配置文件中将 `max_frames` 从 16 改为 8；脚本会自动降低 fps 重试 |
| `CUDA out of memory`（VideoMAE） | 16 帧占用较高 | 减少 `num_frames`（如改为 8），注意需与模型要求匹配 |
| `ImportError: No module named 'timm'` | 缺少依赖 | `pip install timm omegaconf` |
| `IOError: 无法打开视频` | 视频路径错误 | 检查配置文件中 `dataset_path` 和 `video_dir` 是否与实际目录一致 |
| inode 告警 | AutoDL 系统盘 inode 限制 | 将 `output_root` 改到 `/root/autodl-fs`（文件存储，不受 inode 限制）|
| 特征文件形状不对 | 配置误用了旧版脚本 | 确认使用的是 `feature_extractors/` 下的新版脚本 |
