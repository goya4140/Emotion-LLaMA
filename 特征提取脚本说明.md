# Emotion-LLaMA 特征提取脚本说明

> 操作步骤见 [`操作指导.md`](./操作指导.md)，本文档专注于脚本的设计原理与技术细节。

---

## 一、设计思路

本脚本（`extract_emotion_llama_features.py`）与 `emotion_qwen_feature_extractor/` 的设计思路完全类比：

**核心思想**：只加载模型的"编码器"部分，丢弃 LLM 主干，对数据集进行批量特征预提取，实现"提取一次，反复使用"。

| | Qwen 提取器 | LLaMA 提取器（本脚本）|
|---|---|---|
| 完整模型 | Emotion-Qwen | Emotion-LLaMA (MiniGPT-v2) |
| 保留模块 | `vpm + generalcompressor + emotioncompressor + gatenet` | `visual_encoder + ln_vision + llama_proj` |
| 丢弃模块 | `llm + lm_head` | `llama_model`（完全不加载，节省 ~13GB） |
| 输出形状 | `[1344, 3584]` | `[256, 4096]` |
| 输入格式 | 视频（多帧，`process_vision_info`）| 图像帧（OpenCV + BLIP2 transforms）|
| 保存格式 | `.pt`（含 features/metadata）| `.pt`（完全相同格式）|

---

## 二、模型架构与权重来源

Emotion-LLaMA 的视觉侧由三个组件构成，权重来源各不相同：

```
完整 Emotion-LLaMA 模型
│
├── visual_encoder（EVA-CLIP-G）     ← 权重来源：eva_vit_g.pth（BLIP2 预训练，自动下载）
│     └── 参数量：约 10 亿，frozen（训练时冻结，不更新）
│
├── ln_vision（LayerNorm 1408）      ← 权重来源：Emoation_LLaMA.pth（HuggingFace Space）
│
├── llama_proj（Linear 5632→4096）  ← 权重来源：Emoation_LLaMA.pth（同上）
│
└── llama_model（LLaMA-2-7B + LoRA）← 本脚本完全不加载
```

`Emoation_LLaMA.pth`（922MB）只存储了**训练中有参数更新的层**，EVA-CLIP-G 本体因为被冻结，所以单独存放（自动下载）。

## 三、数据流

严格复现 `minigpt_v2.py encode_img()` 方法的视觉分支（第 92-116 行）：

```
输入视频
  │
  ▼ OpenCV 均匀采样帧（fps=1.0，最多 16 帧）
  │
  ▼ BLIP2 预处理（Resize 448×448 + 归一化）
  │
  ▼  image_batch: [N, 3, 448, 448]   ← N = 采样帧数
  │
  ▼ visual_encoder（EVA-CLIP-G）
  │
  ▼  image_feats: [N, 1025, 1408]    ← 1 个 CLS token + 1024 个 patch token
  │
  ▼ ln_vision（LayerNorm）
  │
  ▼  image_embeds: [N, 1025, 1408]
  │
  ▼ 去掉 CLS token（取 [:, 1:, :]）
  │
  ▼  image_embeds: [N, 1024, 1408]
  │
  ▼ reshape（4 个 patch 合并为 1，即 view(N, 256, 5632)）
  │   ← 对应 encode_img 第 103-104 行：
  │     image_embeds.view(bs, int(pn/4), int(hs*4))
  │
  ▼  image_embeds: [N, 256, 5632]
  │
  ▼ llama_proj（Linear 5632 → 4096）
  │   ← 对应 encode_img 第 105 行：
  │     image_inputs_llama = self.llama_proj(image_embeds)
  │
  ▼  frame_features: [N, 256, 4096]
  │
  ▼ 时间维度平均（多帧 → 单向量）
  │   features = frame_features.mean(dim=0)
  │
  ▼  features: [256, 4096]           ← 最终保存的特征
```

`[256, 4096]` 对应 Emotion-LLaMA 送入 LLaMA 的图像 token 序列（共 256 个 token，每个 4096 维）。

## 四、关键参数说明

### 4.1 `use_llama_proj`（配置文件 `model.use_llama_proj`）

| 值 | 输出形状 | 语义 | 适用场景 |
|---|---|---|---|
| `true`（推荐）| `[256, 4096]` | LLaMA embedding 空间的视觉 token | 与 Emotion-LLaMA 训练对齐 |
| `false` | `[1024, 1408]` | EVA-CLIP-G 原始 patch embedding | 更通用，与其他 BLIP2 系模型兼容 |

### 4.2 `fps` 与 `max_frames`

EVA-CLIP-G 是**图像模型**，对"帧"独立处理后取均值。

- `fps=1.0`：每秒取 1 帧，一个 5s 视频取 5 帧 → 平均 → `[256, 4096]`
- `max_frames=16`：上限保护，防止超长视频占用过多显存
- OOM 时自动降低 fps（1.0 → 0.5 → 0.25）重试，与 Qwen 版逻辑一致

### 4.3 归一化参数

使用 BLIP2/CLIP 标准归一化（来自 `blip_processors.py`）：
```python
mean = (0.48145466, 0.4578275, 0.40821073)
std  = (0.26862954, 0.26130258, 0.27577711)
```
与 Emotion-LLaMA 训练时的 vis_processor 完全一致。

## 五、输出格式

每个视频对应一个 `.pt` 文件，格式与 Qwen 提取器**完全相同**：

```
{output_dir}/{dataset_name}/single_files/{video_id}_fps{fps}_features.pt
```

文件内容（Python dict）：

```python
{
    "features":      torch.Tensor,   # 形状 [256, 4096]，dtype=float16
    "feature_shape": torch.Size,     # torch.Size([256, 4096])
    "metadata": {
        "video_id":      str,        # 如 "sample_00000001"
        "video_file":    str,        # 如 "sample_00000001.avi"
        "label":         str,        # 如 "happy"
        "fps":           float,      # 实际使用的 fps（OOM 时可能低于配置值）
        "feature_shape": list,       # [256, 4096]
        "encoder_type":  str,        # "emotion_llama_eva_clip"
    },
    "timestamp":     str,            # 提取时间 "2025-05-27 12:34:56"
}
```

加载方式（与 Qwen 版完全一致）：
```python
from emotion_qwen_feature_extractor.feature_utils import load_features_single

data     = load_features_single("sample_00000001_fps1_features.pt")
features = data["features"]   # torch.Tensor [256, 4096]
label    = data["metadata"]["label"]
```

| 工具函数 | 定义在 `feature_utils.py` | 复用 `feature_utils.py` |

| 断点续传 | ✅ | ✅（相同逻辑）|
| OOM 处理 | 降低 fps 重试 | 降低 fps 重试（相同逻辑）|
| inode 监控 | ✅ | ✅（相同逻辑）|
