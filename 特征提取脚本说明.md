# Emotion-LLaMA 特征提取脚本说明

> 操作步骤见 [`操作指导.md`](./操作指导.md)，本文档专注于三路编码器的设计原理与技术细节。

---

## 一、设计思路

本工具集使用三个**独立**的视觉编码器对视频进行离线特征预提取，不依赖 Emotion-LLaMA 的投影层权重，实现"提取一次，反复使用"。

| 编码器 | 对应脚本 | 输出形状 | 对应投影层（输入边界） |
|--------|---------|---------|----------------------|
| EVA-CLIP-G | `feature_extractors/eva_clip/extract_eva_clip_features.py` | `[1024, 1408]` | `llama_proj (Linear 5632→4096)` 前 |
| MAE-Large | `feature_extractors/mae/extract_mae_features.py` | `[1024]` | `feats_llama_proj1 (Linear 1024→4096)` 前 |
| VideoMAE-Large | `feature_extractors/videomae/extract_videomae_features.py` | `[1024]` | `feats_llama_proj2 (Linear 1024→4096)` 前 |

三个编码器在 Emotion-LLaMA 训练中均为**冻结状态**（`requires_grad=False`），因此直接使用预训练权重即可，无需 Emotion-LLaMA checkpoint。

## 二、EVA-CLIP-G 提取器

### 2.1 模型加载

```python
# vendor/eva_vit.py 中的 create_eva_vit_g（从原架构迁出，无需安装 minigpt4 包）
visual_encoder = create_eva_vit_g(img_size=448, precision="fp16")
ln_vision = LayerNorm(1408)   # FP16 安全 LayerNorm（计算在 float32，结果转 fp16）
# 不加载任何 Emotion-LLaMA checkpoint，不使用 llama_proj / cls_tk_llama_proj
```

权重来源：`eva_vit_g.pth`（约 3.6 GB），首次运行自动从 Google Cloud Storage 下载并缓存到 `~/.cache/torch/hub/checkpoints/`。

### 2.2 数据流

```
输入视频
  │
  ▼ OpenCV 均匀采样（fps=1.0，最多 16 帧）
  │
  ▼ BLIP2 预处理（Resize 448×448，归一化 mean=(0.48145466,0.4578275,0.40821073)）
  │
  ▼ image_batch: [N, 3, 448, 448]   N = 采样帧数
  │
  ▼ visual_encoder（EVA-CLIP-G）
  │
  ▼ image_feats: [N, 1025, 1408]    1 个 CLS + 1024 个 patch token
  │
  ▼ ln_vision（LayerNorm）
  │
  ▼ [N, 1025, 1408]
  │
  ▼ 去掉 CLS token（取 [:, 1:, :]）
  │
  ▼ [N, 1024, 1408]
  │
  ▼ 时间维度均值（多帧 → 单矩阵）
  │
  ▼ features: [1024, 1408]          ← 最终保存的特征
```

`[1024, 1408]` 对应 `llama_proj` 的输入：patch token 序列，1024 个位置，每个 1408 维。

## 三、MAE-Large 提取器

### 3.1 模型加载

```python
from transformers import ViTMAEModel
model = ViTMAEModel.from_pretrained("facebook/vit-mae-large")
model.config.mask_ratio = 0.0  # 推理时禁用随机掩码
```

### 3.2 数据流

```
输入视频
  │
  ▼ OpenCV 均匀采样 N 帧（默认 8 帧，max_frames 可配置）
  │
  ▼ Resize 224×224 + ImageNet 归一化
  │   mean=(0.485, 0.456, 0.406)  std=(0.229, 0.224, 0.225)
  │
  ▼ pixel_values: [N, 3, 224, 224]
  │
  ▼ ViTMAEModel.forward()
  │
  ▼ last_hidden_state: [N, 197, 1024]   1 CLS + 196 patch token
  │
  ▼ 去掉 CLS token（取 [:, 1:, :]）
  │
  ▼ [N, 196, 1024]
  │
  ▼ 时空均值（dim=(0,1)，跨帧+跨 patch）
  │
  ▼ features: [1024]                    ← 最终保存的特征
```

`[1024]` 对应 `feats_llama_proj1` 的输入（Emotion-LLaMA 中 `video_features[:, 0, :]`，即 MAE 局部特征分支）。

## 四、VideoMAE-Large 提取器

### 4.1 模型加载

```python
from transformers import VideoMAEModel
model = VideoMAEModel.from_pretrained("MCG-NJU/videomae-large")
```

### 4.2 数据流

```
输入视频
  │
  ▼ OpenCV 均匀采样 16 帧（不足则循环填充）
  │
  ▼ 每帧 Resize 224×224 + ImageNet 归一化
  │
  ▼ pixel_values: [1, 16, 3, 224, 224]
  │
  ▼ VideoMAEModel.forward()
  │
  ▼ last_hidden_state: [1, 1568, 1024]
  │   时空 token 数 = (16/2) × (224/16)² = 8 × 196 = 1568
  │   （tube_size=2，patch_size=16）
  │
  ▼ 时空均值（dim=1）
  │
  ▼ features: [1024]                    ← 最终保存的特征
```

`[1024]` 对应 `feats_llama_proj2` 的输入（Emotion-LLaMA 中 `video_features[:, 1, :]`，即 VideoMAE 时序特征分支）。

## 五、共同特性

### 5.1 OOM 自动恢复（EVA-CLIP）

EVA-CLIP 提取器在遇到显存不足时自动降低 fps 重试：
```
fps: 1.0 → 0.5 → 0.25（最低值，仍失败则跳过该视频）
```

### 5.2 断点续传

所有提取器均支持断点续传，基于双重保险：
1. checkpoint JSON 文件记录已完成的 `video_id`
2. 输出 `.pt` 文件存在性检查

### 5.3 归一化参数对比

| 编码器 | 归一化标准 | mean | std |
|--------|----------|------|-----|
| EVA-CLIP | BLIP2/CLIP | `(0.48145466, 0.4578275, 0.40821073)` | `(0.26862954, 0.26130258, 0.27577711)` |
| MAE | ImageNet | `(0.485, 0.456, 0.406)` | `(0.229, 0.224, 0.225)` |
| VideoMAE | ImageNet | `(0.485, 0.456, 0.406)` | `(0.229, 0.224, 0.225)` |

## 六、输出格式

每个视频对应一个 `.pt` 文件，存储路径：

```
{output_root}/{ENCODER}/{dataset_name}/{video_id}_features.pt
```

文件内容（Python dict）：

```python
{
    "features":      torch.Tensor,   # 形状见上表，dtype=float32（CPU）
    "feature_shape": list,
    "metadata": {
        "video_id":      str,        # 如 "sample_00000001"
        "video_file":    str,        # 如 "sample_00000001.avi"
        "label":         str,        # 如 "happy"
        "encoder_type":  str,        # "EVA-CLIP-G" / "MAE-Large" / "VideoMAE-Large"
        "feature_shape": list,
        # EVA-CLIP 额外包含 fps、num_frames
        # MAE 额外包含 image_size
        # VideoMAE 额外包含 num_frames
    },
    "timestamp": str,                # 提取时间
}
```

加载方式：
```python
import torch
data = torch.load("sample_00000001_features.pt", map_location="cpu", weights_only=False)
features = data["features"]      # torch.Tensor，形状如上
label    = data["metadata"]["label"]
```

## 七、与旧版 LLaMA 提取器对比

旧版 `extract_emotion_llama_features.py`（历史存档）使用 `llama_proj` 投影层，输出 `[256, 4096]`，已被本工具集取代。新版不加载投影层权重，输出向量在投影层**输入**边界，格式更通用，可直接传入 Emotion-LLaMA 原始架构对应的线性层。

| 对比项 | 旧版（extract_emotion_llama_features.py） | 新版（feature_extractors/）|
|-------|----------------------------------------|--------------------------|
| 投影层 | 包含 llama_proj（需要 Emoation_LLaMA.pth）| 不加载，仅使用编码器本体 |
| EVA-CLIP 输出 | `[256, 4096]`（经投影） | `[1024, 1408]`（投影前） |
| MAE/VideoMAE | 未实现 | `[1024]` 各自独立提取 |
| 权重依赖 | eva_vit_g.pth + Emoation_LLaMA.pth | 仅 eva_vit_g.pth（EVA-CLIP）；其余从 HuggingFace 下载 |
