# Emotion-LLaMA 视觉特征提取操作指导

## 概述

本工具从视频数据集中提取 **Emotion-LLaMA 视觉编码器**（EVA-CLIP-G）的中间特征，供下游任务使用。

与 `emotion_qwen_feature_extractor/` 完全类比：

| 对比项 | Qwen 提取器 | LLaMA 提取器（本工具）|
|-------|-----------|----------------|
| 模型 | Emotion-Qwen | Emotion-LLaMA (EVA-CLIP-G) |
| 保留模块 | `vpm + compressors + gatenet` | `visual_encoder + ln_vision + llama_proj` |
| 输出形状 | `[1344, 3584]` | `[256, 4096]`（use_llama_proj=True）|
| 保存格式 | `.pt` 含 features/metadata | `.pt` 含 features/metadata（完全一致）|
| 音频特征 | 无 | 无 |

输出特征保存路径示例：
```
{output_dir}/{dataset_name}/single_files/sample_00000001_fps1_features.pt
```

## 二、模型权重准备

### 2.1 EVA-CLIP-G 权重（自动下载）

脚本调用 `create_eva_vit_g()` 时会**自动下载**并缓存 EVA-CLIP-G 权重：

```
下载地址：https://storage.googleapis.com/sfr-vision-language-research/LAVIS/models/BLIP2/eva_vit_g.pth
缓存位置：~/.cache/torch/hub/checkpoints/eva_vit_g.pth（约 3.6GB）
```

**离线环境预下载：**
```bash
# 方法一：wget 预下载
wget -P ~/.cache/torch/hub/checkpoints/ \
  https://storage.googleapis.com/sfr-vision-language-research/LAVIS/models/BLIP2/eva_vit_g.pth

# 方法二：设置自定义缓存目录
export TORCH_HOME=/root/autodl-tmp/torch_cache
```

### 2.2 Emotion-LLaMA 微调 checkpoint

从 HuggingFace Space 下载 `minigptv2_checkpoint.pth`：

```
HuggingFace Space：https://huggingface.co/spaces/ZebangCheng/Emotion-LLaMA/tree/main
文件名：minigptv2_checkpoint.pth（约 800MB）
存放路径：/root/autodl-tmp/checkpoints/minigptv2_checkpoint.pth
```

**注意：** 此 checkpoint 包含 `ln_vision` 和 `llama_proj` 的微调权重，
**不包含** EVA-CLIP-G 和 LLaMA-2 的基础权重（它们分别自动下载/单独存储）。

### 2.3 LLaMA-2 权重（⚠️ 本工具无需）

本工具采用"仅 Encoder"策略，**不需要** LLaMA-2-7B 权重（~13GB）。
EVA-CLIP + checkpoint 中的 ln_vision/llama_proj 权重已足够。

## 三、配置文件修改

编辑 `feature_extraction_config_llama.yaml`，重点修改以下路径：

```yaml
model:
  ckpt: "/root/autodl-tmp/checkpoints/minigptv2_checkpoint.pth"  # ← 修改为实际路径
  device: "cuda:0"          # ← 根据 GPU 情况调整
  torch_dtype: "float16"    # float16 节省显存，float32 精度更高
  use_llama_proj: true       # true → [256,4096]；false → [1024,1408]

vision_preprocess:
  fps: 1.0       # ← 视频特别短可调高（如 2.0），长视频保持 1.0
  max_frames: 16 # ← 显存不足时降低（如 8）

data:
  dataset_path: "/root/autodl-tmp/datasets/CA-MER"  # ← 修改为实际数据集路径
  annotation_file: "video-aligned.json"              # ← 修改为实际标注文件名
  video_dir: "video-aligned"                         # ← 修改为视频子目录

feature_saving:
  output_dir: "/root/autodl-tmp/features/EmotionLlamaEncoder"  # ← 修改为输出路径

logging:
  output_dir: "/root/autodl-tmp/logs/llama_features"  # ← 日志目录

resume:
  checkpoint_file: "/root/autodl-tmp/logs/llama_feature_checkpoint.json"
```

## 四、运行方式

### 4.1 单数据集运行

```bash
cd /root/autodl-tmp/Emotion-LLaMA

# 使用配置文件默认参数
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml

# 命令行覆盖数据集路径和输出目录
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml \
  --dataset /root/autodl-tmp/datasets/CA-MER \
  --output-dir /root/autodl-tmp/features/EmotionLlamaEncoder
```

### 4.2 多数据集批量运行

```bash
#!/bin/bash
# batch_extract.sh
DATASETS=("CA-MER" "MER2025-ovmer")

for DATASET in "${DATASETS[@]}"; do
  echo "========== 处理数据集: $DATASET =========="
  python extract_emotion_llama_features.py \
    --config feature_extraction_config_llama.yaml \
    --dataset /root/autodl-tmp/datasets/$DATASET \
    --output-dir /root/autodl-tmp/features/EmotionLlamaEncoder
done
```

### 4.3 指定不同的标注文件和视频目录

```bash
# EMER 数据集（MER2025-ovmer），视频路径从 JSON 中读取
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml \
  --dataset /root/autodl-tmp/datasets/MER2025-ovmer \
  --annotation-file "annotation.json" \
  --video-dir "video"

# CA-MER 数据集
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml \
  --dataset /root/autodl-tmp/datasets/CA-MER \
  --annotation-file "video-aligned.json" \
  --video-dir "video-aligned"
```

### 4.4 断点续传

```bash
# 中断后重新运行，会自动跳过已处理的视频（通过 checkpoint.json）
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml

# 强制重新处理（删除 checkpoint 文件）
rm /root/autodl-tmp/logs/llama_feature_checkpoint.json
python extract_emotion_llama_features.py \
  --config feature_extraction_config_llama.yaml
```


## 八、常见问题排查

### 8.1 CUDA Out of Memory

**症状：** `CUDA out of memory` 错误，脚本自动降低 fps 重试

**解决方案：**
1. 降低配置文件中的 `vision_preprocess.max_frames`（如 16→8→4）
2. 降低 `vision_preprocess.fps`（如 2.0→1.0→0.5）
3. 若 RTX 3090（24GB）仍 OOM，检查是否有其他进程占用 GPU：`nvidia-smi`

```yaml
# 低显存配置（8-16GB VRAM）
vision_preprocess:
  fps: 0.5
  max_frames: 8
```

### 8.2 EVA-CLIP 权重下载失败

**症状：** `ConnectionError` 或 `timeout`

**解决方案：**
```bash
# 手动下载后复制到缓存目录
wget --no-check-certificate \
  https://storage.googleapis.com/sfr-vision-language-research/LAVIS/models/BLIP2/eva_vit_g.pth \
  -O ~/.cache/torch/hub/checkpoints/eva_vit_g.pth

# 或使用代理
export https_proxy=http://your-proxy:port
```

### 8.3 导入 minigpt4 失败

**症状：** `ImportError: No module named 'minigpt4'`

**解决方案：**
```bash
# 确认 Emotion-LLaMA-main 目录存在
ls /root/autodl-tmp/Emotion-LLaMA/Emotion-LLaMA-main/minigpt4/

# 确认依赖包已安装
pip install omegaconf timm transformers peft
```

### 8.4 checkpoint 中找不到 ln_vision/llama_proj 权重

**症状：** 日志出现 `⚠️ checkpoint 中未找到 ln_vision 权重`

**排查：**
```python
import torch
ckpt = torch.load("/root/autodl-tmp/checkpoints/minigptv2_checkpoint.pth", map_location="cpu")
state = ckpt.get("model", ckpt)
# 查看所有 key
visual_keys = [k for k in state.keys() if "ln_vision" in k or "llama_proj" in k]
print(visual_keys)
```

如果 key 为空，说明 checkpoint 格式不匹配，检查是否使用了正确的 Emotion-LLaMA 版本。

### 8.5 视频无法读取

**症状：** `IOError: 无法打开视频文件`

**解决方案：**
```bash
# 检查 OpenCV 是否支持该格式
python -c "import cv2; print(cv2.getBuildInformation())" | grep -i "ffmpeg\|gstreamer"

# 安装完整版 OpenCV（含视频解码支持）
pip install opencv-python  # 而非 opencv-python-headless
```

### 8.6 inode 不足

**症状：** `No space left on device` 但磁盘空间充足

**解决方案：**
```bash
# 检查 inode 使用情况
df -i .

# 清理 Qwen 提取器已生成的小文件（若不再需要）
python emotion_qwen_feature_extractor/check_inode_usage.py
```

## 九、与 Qwen 提取器的对比

| 对比项 | Qwen 提取器 | LLaMA 提取器（本工具）|
|-------|-----------|----------------|
| 模型架构 | Emotion-Qwen VL | Emotion-LLaMA (MiniGPT-v2) |
| 视觉编码器 | Emotion_QwenVisionTransformer | EVA-CLIP-G (ViT-G/14) |
| 压缩模块 | MoECompressor × 2 + GateNet | 无（直接 reshape + Linear）|
| 输出形状 | `[1344, 3584]` | `[256, 4096]` |
| 输入格式 | 视频（多帧，qwen_vl_utils）| 图像帧（OpenCV + BLIP2 transforms）|
| 保存格式 | `.pt` | `.pt`（完全一致）|
| 加载代码 | `feature_utils.load_features_single()` | 相同函数 |
| 关键依赖 | `qwen_vl_utils`, `transformers` | `timm`, `omegaconf`, `peft` |

## 十、输出特征在 Emotion-LLaMA 训练中的对接

本工具提取的特征（`[256, 4096]`）对应 `minigpt_v2.py encode_img()` 中的 `image_inputs_llama`，
可直接用于需要视觉特征的下游任务。

若需要对接原始 Emotion-LLaMA 训练流程（first_face.py 数据集），
还需要将本工具提取的特征与 HuBERT 音频特征（`HL-UTT/`）和 VideoMAE 特征（`maeV_399_UTT/`）结合使用。
